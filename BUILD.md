# Build instructions (for AMO reviewers)

This document explains how to reproduce the exact packaged Firefox add-on from
this source. The first-party source (everything under `chrome/popup`,
`chrome/offscreen`, `chrome/background`, `chrome/workspace`, `chrome/_locales`)
is plain, human-readable code — it is **not** transpiled or bundled. Only the
vendored third-party libraries are pre-minified (see "Third-party" below).

## Build environment

- **Operating system:** macOS, Linux, or Windows. (On Windows use WSL or Git
  Bash so the `zip` step works; the `dist/` output itself needs no `zip`.)
- **Node.js:** version **18 or newer** (developed/tested with Node 25). npm ships
  with Node. Install from https://nodejs.org (or via nvm).
- **`zip` command-line utility:** only needed for the final `.zip` packaging
  step (preinstalled on macOS/Linux). Producing the unpacked `firefox/dist/` does
  not require it.

No other global tools are required. The only build dependencies are three npm
dev-packages (Terser, clean-css, html-minifier-terser), installed in the next
step from `chrome/package.json`.

## Steps

```bash
cd chrome
npm install                 # installs terser, clean-css, html-minifier-terser
node build.js --firefox     # builds the Firefox add-on
```

This produces:

- `firefox/dist/` — the unpacked add-on (load via about:debugging → Load
  Temporary Add-on → `firefox/dist/manifest.json`).
- `release/generate-ai-images-firefox_<version>.zip` — the packaged add-on, i.e.
  **the exact file uploaded to AMO**.

The Chrome build (`node build.js`, no flag) is analogous and not needed for
review.

### How the build works

`chrome/build.js` is the single build script. It:
- minifies first-party JS with Terser, CSS with clean-css, HTML with
  html-minifier-terser, and strips whitespace from JSON;
- copies WASM / images / the third-party `vendor/` bundles verbatim;
- overlays the Firefox-specific files from `firefox/` (manifest + background
  page) on top of the shared `chrome/` sources;
- stamps the version from `chrome/package.json` into the manifest;
- zips the result into `release/`.

The add-on version is defined in **one** place: `chrome/package.json`
(`"version"`). It is injected into the manifest at build time.

## Third-party libraries (pre-built, unmodified upstream)

These are vendored as their official minified distributions and are **not**
modified by us:

- **onnxruntime-web** — `chrome/offscreen/vendor/ort*.{mjs,wasm}`
  (https://github.com/microsoft/onnxruntime)
- **Transformers.js** — `chrome/offscreen/vendor/transformers.min.js`
  (https://github.com/huggingface/transformers.js)
- **JSZip** — `chrome/workspace/vendor/jszip.min.js`
  (https://github.com/Stuk/jszip)

Any `Function`-constructor / dynamic-`import` linter warnings originate inside
these bundles. No first-party code uses `eval`, the `Function` constructor, or
dynamic `import()` with a computed value.

## Source layout

- `chrome/` — shared, human-readable source + `build.js`.
- `firefox/` — Firefox overlay (MV3 manifest + background page).
- `chrome/tests/` — automated tests (not part of the shipped add-on).
- `release/`, `*/dist/`, `node_modules/` — build output / deps; **not** included
  in this source archive and regenerated by the steps above.
